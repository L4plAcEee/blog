发表于2025-04-04|更新于2025-04-20|最佳实践
|总字数:1.9k|阅读时长:6分钟|浏览量:

引言：

一份任务，我需要写一份关于实习的总结报告底稿。
这应该可以用 LLM 直接生成一份，但是我并不想这么做，
思维在指尖流动，所以诞生了文字，文字本没有想法，却被人赋予了灵魂。
我想借此去重新组织自己的思考，而不是当个枯燥无味的 Prompt Typer。
这让我重新的去思考：遣词造句，而后组织一篇文章的逻辑。
“像 LLM 一样思考！”我想，我应该如此。

从需求分析 - 到思考限制

需求是一个基本衡量角度，无论是小到当一个市井摊贩，还是大到上市公司总裁，
还是单单只是想把握好自己人生的方向标，需求分析都必不可少。
写文章亦是如此，所以这篇文章的需求是什么呢？
就这样去思考。
当然，我目前所工作的内容目的很明确：
生产一份 学校要求的 关于 实习工作内容的 报告/发言底稿，
这是一个很明确且正式的要求。
那么需求就很明显了：

  1. 阐述 实习工作内容（换句话来说，就是日常是如何工作的？）
  2. 阐述 实习工作经历（更像是要说一些独到经历，毕竟光说日常过于寡淡）
  3. 阐述 工作内容与学校课程知识点的联系（更像是限制？）
  4. 场面话，客套话。

似乎就这么多吧？
在阐述需求的时候，似乎限制也自然出现了，就好像需求和限制天然一体两面一般，
我想也应该如此，需求和限制本就相辅相成，相互匹配。

所以实战必不可少，既然需求已经阐明，那为什么不马上去做？
