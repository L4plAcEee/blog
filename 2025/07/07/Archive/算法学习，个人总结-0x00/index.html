写算法题也快半年了，纯属个人总结

Trap

  1. 当你使用 erase(it) 删除当前元素之后，it 迭代器会失效，因此下一轮循环中 it 的初始位置会出错。你需要在 erase 之后立即更新 it 到下一个合法位置。
    -> it = a.erase(it);

  2. C中的函数传值数组最好另外传 size，因为数组在sizeof会退化为指针，就只会返回数组大小了。

  3. 在长期累乘中，double 类型会出现一定的误差。建议：

    • 在判断 money <= 0 时使用一个小的 epsilon 容忍误差：
      if (money <= 1e-6) return true;

  4. C++万能头bits/stdc++.h似乎只在 clang 和 gcc 有。

  5. 找到了合法解却没有停止循环，导致解被覆盖，需要注意。

  6. IO流处理’\n’注意点:
    cin 和 stringstream 读取时，默认会自动跳过换行符，
    除非明确要求捕获或处理。你想要用 '\n' 来做行计数，
    但实际上 cin 或 stringstream 处理输入时，
    并不会将换行符 '\n' 存入到字符串中。

  7. 尽量不要将浮点数和整型混合运算。

Trick

  1. 组合数学C(x, k)

  2. 矩阵旋转可以分解为转置和翻转

  3. 通过在 main 前构造一个对象/结构体，以达到程序入口时自动初始化的效果。
    算是cpp的语法糖吧，可以组合结构体/对象定义和构造。

  4. 利用字符串逻辑上处理变长参数。
    读取一行输入的第一个字段，判断其类型，从而决定是否继续读取两个数字（3 个参数）或一个数字（2 个参数）。
    不像函数级别的可变参数（如 ... 和 va_list），而是通过输入预处理逻辑+变量缓存（lastOp） 来完成多种格式兼容的。

  5. 用 isalnum() 处理字母和数的判断更为优雅。

  6. 直接 强转size_t 虽然不安全，但是简短

  7. transform 和 for_each 各有优劣吧，for_each更注重原地操作，另一个相反。

  8. 比手搓欧式距离函数更优雅的写法 hypot(x, y):
    hypot 是一个用于计算直角三角形斜边长度的数学函数，
    这个函数之所以叫 hypot，是源自希腊语 hypotenuse（斜边） 的缩写或简写形式。

  9. 一个集合的每个元素在所有子集中会出现 $2^{n-1}$ 次（如果集合大小是 $n$）。

  10. fixed 和 setprecision(int) 配套使用设置IO流输出精度。

  11. 简单寻找素数优化
    若 i = a * b，且 a ≤ b，那么必然有 a ≤ sqrt(i)，
    所以，如果 i 有任何因数，它的最小因数必 ≤ sqrt(i)，因此，检查到sqrt(i)` 就足够了，后面的约数是对称的。
    进阶需要用到 筛。

  12. 连续读取可以使用while(cin >> temp)的写法。

  13. 区间拓展：端点对齐，向右拓展，中断结算。

  14. 如何设置格式:
    使用 iomanip 中的 setw(3) 设置每个数字占 3 格宽度，自动右对齐。
    不需要额外补空格或对齐，setw 自动处理。

  15. 用 dir数组封装 相对位置 是一种优雅的写法。
