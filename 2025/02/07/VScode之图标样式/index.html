VScode之图标样式

发表于2025-02-07|更新于2025-02-09|生活中的观察
|总字数:1.6k|阅读时长:5分钟|浏览量:

前言：

我发现了一个很有趣的事情，当我的VSCODE中md文件被命名为CHANGELOG时，
图标变成了一个时钟样式，而原本的md文件图标样式应该是一个向下的箭头。
我猜想应该是VScode的主题配置的问题。
然后我仔细回想了一下，确实，实际上VScode有很多我之前未曾在意的小细节
譬如：

  • **README.md**：通常使用一个书本或文档图标，因为它是项目的说明文件，描述项目的目的和使用方式。

  • **.gitignore**：为 .gitignore 文件通常使用一个斜杠图标，表示它是 Git 忽略文件，用来告诉 Git 哪些文件不需要被跟踪。

  • **package.json**：为 Node.js 项目的 package.json 文件，VS Code 会显示一个与包管理相关的图标，表示它是包含项目依赖和配置的文件。

  • **.editorconfig**：为 .editorconfig 文件，VS Code 会使用一个齿轮或配置图标，表示该文件用于编辑器设置。

  • **Dockerfile**：为 Dockerfile 文件，通常会显示一个与容器或 Docker 相关的图标。

  • .git 目录：VS Code 会为 .git 目录提供 Git 版本控制图标，表示该目录包含 Git 配置和信息。

  • **LICENSE 或 LICENSE.txt**：VS Code 会为许可文件（如 LICENSE 文件）分配一个文件图标，通常表示这是一个包含许可信息的文件。

许多 VS Code 的图标主题（如 “Material Icon Theme” 或 “VSCode Icons”）会为常见的文件类型提供自定义图标。CHANGELOG 文件通常会被认为是项目的历史记录文件，因此会使用时钟样式的图标来表示

VS Code 的文件和文件夹图标样式实际上由所选择的“图标主题”（Icon Theme）决定。默认情况下，VS Code 使用的是 Seti 图标主题，但也有许多第三方主题（如 VS Code Icons、Material Icon Theme、Fluent Icons 等）提供了不同风格的图标。

1. 代码文件图标

  • JavaScript (.js)
    – 通常使用一个带有“JS”字样的黄色或橙色图标。

  • TypeScript (.ts)
    – 类似 JavaScript，但会显示“TS”字样，颜色和细节上有所区别。

  • Python (.py)
    – 常见的是一个带有 Python 蛇形或 Python 标志的小图标。

  • Java (.java)
    – 通常以咖啡杯或类似符号作为标识。

  • C/C++ (.c, .cpp)
    – 显示带有 C 或 C++ 标识的图标，颜色和设计上会有所区分。

  • C# (.cs)
    – 带有 C# 标识的图标，通常颜色和设计与其他语言区分开来。

  • PHP (.php)
    – 经常使用带有大象标识的图标，因为 PHP 的吉祥物是大象。

2. 标记和样式文件

  • HTML (.html)
    – 图标常以一对尖括号 “< >” 来表现，代表标记语言。

  • CSS (.css)
    – 一般以代表样式的符号显示，颜色上也会与 HTML 区别。

  • Markdown (.md)
    – 常用一个文档或带有大写 “M” 的图标。
    – 例如，README.md 通常有专属的文档图标，而 CHANGELOG 文件（无扩展名或扩展名为 .md）有时会显示为带时钟的图标，暗示版本历史或时间记录。

3. 配置和数据文件

  • package.json
    – 显示一个与 Node.js/npm 相关的图标，通常带有特殊的包管理标识。

  • tsconfig.json、.editorconfig
    – 配置文件通常使用齿轮或设置类图标，表示这是项目配置文件。

  • .gitignore
    – 常见地显示一个 Git 或版本控制相关的图标，有时会是一个带有“否定”标记的文件图标。

  • Dockerfile
    – 通常使用 Docker 鲸鱼图标，表明这是一个用于构建容器镜像的文件。

  • 其他 JSON、YAML、XML 文件
    – 一般会以简洁的“{}”（对于 JSON）或相应标识显示，突出数据结构特点。

4. 数据、日志与脚本文件

  • SQL (.sql)
    – 数据库脚本文件通常使用带有数据库符号（如表格或关系图）的图标。

  • CSV、TSV (.csv, .tsv)
    – 常以表格、电子表格形式的图标显示，突出数据的结构性。

  • 日志文件 (.log)
    – 一般使用文档或纸张样式的图标，有时会带有代表“记录”的小图标。

  • Shell 脚本 (.sh)、批处理文件 (.bat)
    – 脚本文件通常显示终端或命令行窗口的图标。

5. 资源文件

  • 图像文件 (.png, .jpg, .gif, .svg)
    – 通常显示一个小相片或预览图标；SVG 文件可能有专门的矢量图标。

  • 音频和视频文件 (.mp3, .mp4 等)
    – 分别有音乐符号或视频播放按钮的图标。

6. 文件夹图标

  • 默认文件夹
    – 显示为常规文件夹图标（闭合或打开状态会有不同的样式）。

  • 特殊文件夹
    – 例如 node_modules 文件夹通常会有 Node.js 标志；.git 文件夹会显示 Git 标识。
    – 源码文件夹（如 src）和测试文件夹（如 test 或 tests）在某些主题中也可能有专门的图标。

7. 其他常见文件

  • LICENSE、NOTICE
    – 这些通常显示一个文档图标，有时带有法律、公文或合同的象征。

  • README
    – 显示为说明文档的图标，通常设计简洁、易辨认。

  • CHANGELOG
    – 如你所发现的，这类文件有时会显示成带时钟、历史记录或日历样式的图标，暗示文件中记录了项目的更新和版本历史。

8. 图标主题举例

  • Seti Icons
    – VS Code 默认的图标主题，图标种类丰富，风格简洁。

  • VS Code Icons
    – 一个第三方图标主题，提供了更为详细和自定义的图标，适用于多种文件类型和项目文件。

  • Material Icon Theme
    – 基于 Material Design 风格的图标主题，图标风格现代简洁。

  • Fluent Icons
    – 采用微软 Fluent 设计语言，图标风格统一且符合 Windows 设计理念。

VS Code 中常见的图标样式主要分为代码文件、配置文件、数据与日志文件、资源文件、文件夹以及其他特殊文件。不同的图标主题可能会对同一类型的文件使用不同的图标样式。

总结：

实际上，通过对图标的认知，其实可以很快的认识到前人们对文件命名的最佳实践，
为什么这么说，因为之所以为他们添加默认的特殊图标样式，就是因为已经成为了人们的共识
所以大概率是最佳实践，因此在我此后的实践中，就可以使用这些最佳实践。
